Reporting a Vulnerability
We take the security of our software seriously. If you believe you have found a security vulnerability, please report it to us responsibly so we can resolve the issue before it is made public. 
How to report
Please do not report security vulnerabilities via public GitHub issues. Instead, send a detailed report to [Insert Email Address/Link]. 
Include the following in your report:
A description of the vulnerability.
The version(s) of the software affected.
Steps to reproduce the issue (proof-of-concept code or screenshots are highly appreciated). 
What to expect
Acknowledgment: You will receive an automated or manual acknowledgment of your report within [e.g., 48 hours].
Updates: We will provide status updates at least once every [e.g., week] until the issue is resolved.
Resolution: If the vulnerability is accepted, we will work on a fix and coordinate a release date with you.
Public Disclosure: Once a fix is released, we will credit you for the discovery (if desired) in our changelog or security advisory. 
Policy for Declined Reports
If we determine that the reported issue does not pose a security risk or is intended behavior, we will provide a detailed explanation as to why the report was declined.
