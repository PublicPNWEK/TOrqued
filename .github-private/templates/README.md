# Private Templates

This directory contains internal issue and pull request templates for organization use.

## Purpose

Store templates here for:
- Internal bug reports
- Security vulnerability reports
- Internal feature requests
- Organization-specific processes

## Template Types

### Issue Templates
Create templates for different types of internal issues:
- `internal-bug-report.md` - For reporting internal bugs
- `security-report.md` - For reporting security vulnerabilities
- `infrastructure-issue.md` - For infrastructure-related issues

### Pull Request Templates
Create templates for internal PRs:
- `internal-pr-template.md` - Default template for internal PRs
- `security-fix-template.md` - For security-related changes
- `infrastructure-change-template.md` - For infrastructure changes

## Usage

To use these templates in GitHub:

1. Reference them in issue/PR creation
2. Or set up automation to apply them based on labels
3. Or use GitHub's template selection feature

## Best Practices

1. Keep templates concise and focused
2. Include clear instructions
3. Use checklists for required information
4. Provide examples where helpful
5. Update templates based on feedback
